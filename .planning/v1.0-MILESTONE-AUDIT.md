## E2E Flow Verification (1/1)

### Daily Content Pipeline

```
GitHub Actions cron (00:00 UTC / 07:00 WIB) OR workflow_dispatch
  -> python main.py
    -> main(): logging.basicConfig() + load_dotenv() + run_pipeline()
      -> Stage 1: gather_source_material(sheet_id=GOOGLE_SHEET_ID)
           5 sources: pum.nl scraper, RSS feed, content brief YAML,
                      Google Sheets, Gemini web search
           Graceful degradation: individual failures skip source
           RuntimeError only if ALL sources fail (AIGEN-01 guard)
      -> Stage 2: generate_post(source_material) -> GeneratedPost
           Pillar rotation (day % 4), template mapping, Gemini API call,
           structured output parsing, template_data validation
      -> Stage 3: render_image(post) -> image_path
           Dictionary dispatch: template_type -> Template class
           Template.render(template_data) -> 1080x1080 RGBA PNG
           Template.save(img, output/pum_post_{date}.png)
      -> Stage 4: send_post_email(post, image_path)
           format_email_body: bilingual captions, #hashtags, posting suggestion
           compose_email: MIMEMultipart with PNG attachment
           send_email: Gmail SMTP (smtp.gmail.com:587, STARTTLS, App Password)
    -> sys.exit(0) on success, sys.exit(1) on failure
```

**Status:** Complete — all stages wired, tested (12 offline tests), and CI-integrated.

## Tech Debt (3 items, non-blocking)

### Phase 3: AI Content Generation
- **AIGEN-05 semantic nuance:** Template type is deterministically mapped via `PILLAR_TEMPLATE_MAP`, not freely selected by Gemini. Achieves the correct outcome (right template per pillar) but diverges from requirement wording. `generator.py:97` validates `result.template_type` without asserting it matches the instructed value — low-probability unguarded path.

### Phase 4: Content Research Sources
- **CWD-relative content brief path:** `gather_source_material` defaults `content_brief_path="content_brief.yaml"` (relative). Resolves correctly when run from project root (CI checkout), but would silently skip if CWD differs. Graceful degradation prevents pipeline failure.

### Phase 1: Foundation & Brand Setup
- **Icon filename typo:** `brand_config.yaml` maps `water_waste` to `"Watr, Waste & Environment.png"` — the actual file matches this spelling, so no functional break. Cosmetic only.
