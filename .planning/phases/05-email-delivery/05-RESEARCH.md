# Phase 5: Email Delivery - Research

**Researched:** 2026-03-01
**Domain:** Email composition (MIME), SMTP transport, Gmail App Password authentication
**Confidence:** HIGH

## Summary

Phase 5 delivers the generated Instagram post (image + caption + hashtags + posting suggestion) to the PUM Indonesia team via email, ready to copy-paste and post. The entire stack is Python standard library: `smtplib` for SMTP transport and `email.mime` for message construction. No third-party email libraries are needed.

The upstream data contract is already well-defined. `content_generator.schemas.GeneratedPost` provides `caption_id` (Bahasa Indonesia), `caption_en` (English), `hashtags` (list of strings), `posting_suggestion` (time and theme), `content_pillar`, and `template_type`. The image is a 1080x1080 PNG generated by the `templates` package and saved to `output/`. The email module receives these two artifacts -- a `GeneratedPost` dataclass and a file path to the PNG image -- and composes + sends the email.

Gmail SMTP with App Password authentication is confirmed working as of 2026. The setup requires 2-Step Verification on the Google account, then generating a 16-character App Password. The SMTP connection uses `smtp.gmail.com:587` with STARTTLS. The project's `.env.example` already defines the three required environment variables: `GMAIL_ADDRESS`, `GMAIL_APP_PASSWORD`, and `RECIPIENT_EMAIL` (user confirmed recipient is `awthedev@gmail.com`).

**Primary recommendation:** Use Python standard library only (`smtplib` + `email.mime`). Build two modules: (1) `email_sender/smtp_client.py` -- Gmail SMTP connection, authentication, and send with retry; (2) `email_sender/composer.py` -- email template composition with MIMEMultipart, plain text body with bilingual captions/hashtags/metadata, and PNG image attachment. No new pip dependencies required.

## Standard Stack

### Core
| Library | Version | Purpose | Why Standard |
|---------|---------|---------|--------------|
| [smtplib](https://docs.python.org/3/library/smtplib.html) | stdlib | SMTP client for sending email via Gmail | Python standard library; no installation needed; full SMTP protocol support including STARTTLS and authentication |
| [email.mime](https://docs.python.org/3/library/email.mime.html) | stdlib | Constructing MIME multipart email with text body and image attachment | Python standard library; MIMEMultipart, MIMEText, MIMEImage cover all requirements; automatic Content-Type and encoding |
| [ssl](https://docs.python.org/3/library/ssl.html) | stdlib | TLS context for secure SMTP connection | `ssl.create_default_context()` provides secure defaults; loads system CA certs; enables hostname checking |

| Library | Version | Purpose | When to Use |
|---------|---------|---------|-------------|
| [email.utils](https://docs.python.org/3/library/email.utils.html) | stdlib | `formatdate()` for RFC 2822 date headers, `formataddr()` for proper From/To formatting | Always -- ensures email headers conform to standards |

### Pattern 1: SMTP Client with Context Manager
**What:** Wrap Gmail SMTP connection in a function that handles connect, STARTTLS, login, send, and cleanup. Use `with` statement for automatic QUIT.
**When to use:** Every email send operation.
**Example:**
```python
# Source: https://docs.python.org/3/library/smtplib.html
import smtplib
import ssl
import os
import logging

logger = logging.getLogger(__name__)

GMAIL_SMTP_HOST = "smtp.gmail.com"
GMAIL_SMTP_PORT = 587

def send_email(msg):
    """Send a composed email message via Gmail SMTP.

    Args:
        msg: email.mime.multipart.MIMEMultipart message object.

    Raises:
        ValueError: If GMAIL_ADDRESS or GMAIL_APP_PASSWORD not set.
        smtplib.SMTPAuthenticationError: If credentials are invalid.
        smtplib.SMTPException: If send fails.
    """
    gmail_address = os.environ.get("GMAIL_ADDRESS")
    gmail_password = os.environ.get("GMAIL_APP_PASSWORD")

    if not gmail_address or not gmail_password:
        raise ValueError(
            "GMAIL_ADDRESS and GMAIL_APP_PASSWORD environment variables required"
        )

    context = ssl.create_default_context()

    with smtplib.SMTP(GMAIL_SMTP_HOST, GMAIL_SMTP_PORT, timeout=30) as server:
        server.starttls(context=context)
        server.login(gmail_address, gmail_password)
        server.send_message(msg)
        logger.info("Email sent successfully to %s", msg["To"])
```

### Pattern 2: Email Composition with MIMEMultipart
**What:** Compose a `MIMEMultipart('mixed')` message with plain text body and PNG image attachment. The body contains copy-paste-ready bilingual captions, hashtags, and posting metadata.
**When to use:** Before every send operation; takes GeneratedPost + image path as input.
**Example:**
```python
# Source: https://docs.python.org/3/library/email.mime.html
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
from email.mime.image import MIMEImage
from email.utils import formatdate, formataddr
from datetime import date

def compose_email(post, image_path, from_addr, to_addr):
    """Compose email with post content and image attachment.

    Args:
        post: GeneratedPost with captions, hashtags, posting_suggestion.
        image_path: Path to generated 1080x1080 PNG image.
        from_addr: Sender Gmail address.
        to_addr: Recipient email address.

    Returns:
        MIMEMultipart message ready for send_email().
    """
    msg = MIMEMultipart("mixed")
    msg["From"] = formataddr(("PUM Content Generator", from_addr))
    msg["To"] = to_addr
    msg["Date"] = formatdate(localtime=True)
    msg["Subject"] = f"PUM Instagram Post - {date.today().isoformat()}"

    # Plain text body with copy-paste sections
    body = format_email_body(post)
    msg.attach(MIMEText(body, "plain", "utf-8"))

    # Image attachment
    with open(image_path, "rb") as f:
        img = MIMEImage(f.read(), _subtype="png")
    img.add_header(
        "Content-Disposition", "attachment",
        filename=f"pum_post_{date.today().isoformat()}.png"
    )
    msg.attach(img)

    return msg
```

### Pattern 3: Copy-Paste-Ready Email Body Format
**What:** Format the email body with clear sections and separators so the team can copy each section independently.
**When to use:** Called by compose_email() to build the body text.
**Example:**
```python
def format_email_body(post):
    """Format GeneratedPost into copy-paste-ready email body.

    Sections:
    1. Content info (pillar, template, date)
    2. Posting suggestion (time and theme)
    3. Bahasa Indonesia caption (ready to copy)
    4. English caption (ready to copy)
    5. Hashtags (ready to copy with # prefix)
    """
    hashtags_str = " ".join(f"#{tag}" for tag in post.hashtags)

    body = f"""PUM Indonesia - Daily Instagram Post
{'=' * 40}

POSTING SUGGESTION
{'-' * 20}
{post.posting_suggestion}

Content pillar: {post.content_pillar}
Template type: {post.template_type}

CAPTION (Bahasa Indonesia)
{'-' * 20}
{post.caption_id}

CAPTION (English)
{'-' * 20}
{post.caption_en}

HASHTAGS
{'-' * 20}
{hashtags_str}

{'=' * 40}
Image attached: pum_post_{date.today().isoformat()}.png

Generated by PUM Content Generator
"""
    return body
```

### Anti-Patterns to Avoid
- **Hardcoded credentials:** Never embed Gmail address or App Password in source code. Always use environment variables.
- **Using regular Gmail password:** Gmail blocks regular password SMTP login. Must use App Password (16-character code from Google Account security settings).
- **HTML email body for v1:** Plain text is simpler, more reliable for copy-paste, and HTML is explicitly v2 scope (DELV2-01).
- **Importing credentials at module level:** Environment variables should be read at call time, not import time. This enables testing with mock values.
- **Skipping `starttls()`:** Gmail SMTP on port 587 requires STARTTLS before authentication. Connection will fail without it.
- **Forgetting `ssl.create_default_context()`:** Passing `context=None` to `starttls()` works but uses less secure defaults. Always create a proper SSL context.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| MIME message construction | Custom string concatenation of email headers/body/attachments | `email.mime.multipart.MIMEMultipart` + `MIMEText` + `MIMEImage` | MIME boundaries, content-type headers, base64 encoding, charset handling are all error-prone; stdlib handles all edge cases |
| Base64 encoding of image attachment | Manual `base64.b64encode()` on image bytes | `MIMEImage` (auto-encodes via `email.encoders.encode_base64`) | MIMEImage automatically encodes binary data and sets correct Content-Transfer-Encoding header |
| Email date formatting | `datetime.now().strftime()` for Date header | `email.utils.formatdate(localtime=True)` | RFC 2822 date format has timezone rules; `formatdate()` handles this correctly |
| SMTP connection lifecycle | Manual socket management with try/finally | `smtplib.SMTP` context manager (`with` statement) | Automatic QUIT command and connection cleanup on exit, even on exceptions |

**Key insight:** Email is a 40-year-old protocol with many edge cases in encoding, headers, and MIME boundaries. The Python standard library handles all of these correctly. Any custom implementation will have subtle bugs.

## Common Pitfalls

### Pitfall 1: App Password Not Generated or 2FA Not Enabled
**What goes wrong:** `SMTPAuthenticationError` (534, 'Please log in via your web browser') when trying to login.
**Why it happens:** Gmail requires 2-Step Verification to be enabled before App Passwords can be generated. Without 2FA, the App Password option doesn't appear in Google Account settings.
**How to avoid:** Document the setup steps: (1) Enable 2-Step Verification at `myaccount.google.com/security`, (2) Go to App Passwords at `myaccount.google.com/apppasswords`, (3) Generate a password for "Mail" on "Other (PUM Content Generator)", (4) Copy the 16-character password (no spaces) to `GMAIL_APP_PASSWORD`.
**Warning signs:** Authentication error on first deployment; App Password option not visible in Google Account.

### Pitfall 2: Forgetting to Set charset='utf-8' on MIMEText
**What goes wrong:** Bahasa Indonesia characters (accented vowels, special characters) get corrupted or replaced with question marks.
**Why it happens:** `MIMEText` defaults to `us-ascii` if the text is pure ASCII, but Bahasa Indonesia captions may contain characters outside ASCII.
**How to avoid:** Always pass `_charset="utf-8"` to `MIMEText()`. This ensures correct encoding regardless of content.
**Warning signs:** Garbled text in received emails; `UnicodeEncodeError` during message construction.

### Pitfall 3: Connection Timeout in CI/CD Environment
**What goes wrong:** `TimeoutError` or `ConnectionRefusedError` when running in GitHub Actions or restricted network.
**Why it happens:** Some CI environments block outbound SMTP connections. GitHub Actions does NOT block SMTP (port 587 is open), but timeouts can occur under load.
**How to avoid:** Set explicit `timeout=30` on `smtplib.SMTP()`. Implement retry with backoff (consistent with project pattern from `generator.py`). Log the error clearly so CI failures are diagnosable.
**Warning signs:** Intermittent failures in CI; consistent success locally but failure in Actions.

### Pitfall 4: Image File Not Found or Empty
**What goes wrong:** `FileNotFoundError` or zero-byte attachment when the image generation step failed silently.
**Why it happens:** The image generation pipeline (Phase 2) might fail but the email step still runs.
**How to avoid:** Validate that the image file exists and has non-zero size before composing the email. Raise a clear error if the image is missing.
**Warning signs:** Email arrives without attachment; email arrives with 0-byte PNG file.

### Pitfall 5: App Password Stored with Spaces
**What goes wrong:** Authentication fails even though the password looks correct.
**Why it happens:** Google displays App Passwords as `xxxx xxxx xxxx xxxx` with spaces for readability, but the actual password is 16 characters without spaces.
**How to avoid:** Strip whitespace from the App Password before use: `gmail_password.replace(" ", "")`. Document that spaces should be removed when storing in environment variable or GitHub Secret.
**Warning signs:** `SMTPAuthenticationError` despite correct-looking credentials.

### Complete Email Send Flow
```python
# Source: https://docs.python.org/3/library/smtplib.html
# Source: https://docs.python.org/3/library/email.mime.html
import smtplib
import ssl
import os
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
from email.mime.image import MIMEImage
from email.utils import formatdate, formataddr
from datetime import date

def send_post_email(post, image_path):
    """Top-level function: compose and send post email.

    Args:
        post: GeneratedPost from content_generator.
        image_path: Path to generated PNG image.

    Raises:
        ValueError: If credentials or recipient not configured.
        FileNotFoundError: If image_path doesn't exist.
        smtplib.SMTPException: If send fails.
    """
    gmail_addr = os.environ.get("GMAIL_ADDRESS")
    gmail_pass = os.environ.get("GMAIL_APP_PASSWORD")
    recipient = os.environ.get("RECIPIENT_EMAIL")

    if not all([gmail_addr, gmail_pass, recipient]):
        raise ValueError(
            "GMAIL_ADDRESS, GMAIL_APP_PASSWORD, and RECIPIENT_EMAIL "
            "environment variables are required"
        )

    # Strip spaces from App Password (Google displays with spaces)
    gmail_pass = gmail_pass.replace(" ", "")

    # Validate image exists
    if not os.path.isfile(image_path):
        raise FileNotFoundError(f"Image not found: {image_path}")

    # Compose message
    msg = MIMEMultipart("mixed")
    msg["From"] = formataddr(("PUM Content Generator", gmail_addr))
    msg["To"] = recipient
    msg["Date"] = formatdate(localtime=True)
    msg["Subject"] = f"PUM Instagram Post - {date.today().isoformat()}"

    # Text body
    body = format_email_body(post)
    msg.attach(MIMEText(body, "plain", "utf-8"))

    # Image attachment
    with open(image_path, "rb") as f:
        img_part = MIMEImage(f.read(), _subtype="png")
    img_part.add_header(
        "Content-Disposition", "attachment",
        filename=f"pum_post_{date.today().isoformat()}.png"
    )
    msg.attach(img_part)

    # Send via Gmail SMTP
    context = ssl.create_default_context()
    with smtplib.SMTP("smtp.gmail.com", 587, timeout=30) as server:
        server.starttls(context=context)
        server.login(gmail_addr, gmail_pass)
        server.send_message(msg)
```

### Environment Variable Pattern (Consistent with Project)
```python
# Matches existing patterns from content_generator/gemini_client.py
# and research_sources/sheets_reader.py
import os

def _get_smtp_credentials():
    """Read SMTP credentials from environment variables.

    Returns:
        Tuple of (gmail_address, gmail_password, recipient_email).

    Raises:
        ValueError: If any required variable is missing.
    """
    gmail_addr = os.environ.get("GMAIL_ADDRESS")
    gmail_pass = os.environ.get("GMAIL_APP_PASSWORD")
    recipient = os.environ.get("RECIPIENT_EMAIL")

    missing = []
    if not gmail_addr:
        missing.append("GMAIL_ADDRESS")
    if not gmail_pass:
        missing.append("GMAIL_APP_PASSWORD")
    if not recipient:
        missing.append("RECIPIENT_EMAIL")

    if missing:
        raise ValueError(
            f"Missing environment variables: {', '.join(missing)}"
        )

    return gmail_addr, gmail_pass.replace(" ", ""), recipient
```

## Open Questions

1. **Gmail daily send limit for App Passwords**
   - What we know: Gmail free accounts allow ~500 emails/day. This project sends 1 email/day, well within limits.
   - What's unclear: Whether Google throttles App Password emails differently from OAuth emails.
   - Recommendation: Not a concern for 1 email/day. Monitor for errors if volume increases in v2.

2. **Email delivery to spam folder**
   - What we know: Emails from Gmail to Gmail (sender and recipient are both @gmail.com) are rarely flagged as spam. The sender is the user's own Gmail account.
   - What's unclear: Whether automated emails via App Password are treated differently by spam filters.
   - Recommendation: Test with a real send during development. If spam-flagged, add the sender to contacts. Not a concern for same-account sends.
